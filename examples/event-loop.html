The mechanism: 32.4s of nothing

JS never gives the request a turn for 32.4s · once scheduled everything is fast → the process is starved, not its dependencies

The cause: O(N²) re-parsing

each record's text parsed once · comparison texts normalized once, not per pair

The fix: banded Levenshtein

0.99 threshold ⇒ distance ≤ 1%·maxLen — band is EXACT for the ≤k decision, verified by fuzz vs full DP

Result: 56s → 54ms

real function, synthetic record lists · laptop, prod container ≈ 2× · byte-identical output
